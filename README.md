# 🏸 羽毛球体能训练 PWA

基于运动科学论文设计的羽毛球专项体能训练计划生成器。每次训练 30 分钟，针对羽毛球运动所需的核心能力进行系统提升。

**在线访问** → https://luca-huang.github.io/badminton/

---

## 设计背景

网上羽毛球训练教程繁杂，缺乏系统性；健身房训练动作与羽毛球专项能力脱节。本项目参考以下运动科学研究，将训练动作与羽毛球实战能力直接挂钩：

- BMC Sports Science：羽毛球体能训练系统性综述
- PLOS ONE：核心训练对羽毛球专项表现的影响
- Nature Scientific Reports：平衡与增强式训练对羽毛球运动员的效果（6周内敏捷提升 1.35s）

---

## 训练模块

| 模块 | 针对能力 | 羽毛球关联 |
|------|----------|------------|
| 核心旋转 | 躯干稳定 + 旋转爆发 | 扣杀、网前推挡的鞭打力 |
| 下肢爆发 | 单腿蹬地力量 | 起跳、急停、变向启动 |
| 多向敏捷 | 横向移动速度 | 四角步法、被动防守还原 |
| 有氧耐力 | 心肺 + 抗疲劳 | 长局体力维持、连续得分 |
| 肩臂专项 | 旋转肌群 + 肩胛稳定 | 扣杀速度、防受伤 |

**训练日**：周一 / 周三 / 周四 / 周六，每次约 30 分钟

## 本地运行

```bash
# 克隆
git clone https://github.com/Luca-Huang/badminton.git
cd badminton

# 启动本地服务（ES Modules 需要 HTTP 服务器）
npx serve .
# 然后访问 http://localhost:3000
```

> 直接双击 `index.html` 无法运行（ES Module 跨域限制）

## MuscleWiki 免费档用法（离线同步）

为避免运行时消耗 API 配额，仓库提供了离线同步脚本：

```bash
# 1) 设置 API Key（不要写入仓库）
export MW_API_KEY='mw_xxxxxxxxxxxxx'

# 2) 同步 30 个动作到本地库（默认含 30 天缓存）
node scripts/sync-mw-library.mjs --out data/exercises.mw.json
```

- 输出文件：`data/exercises.mw.json` 和 `data/exercises.mw.js`
- 前端默认读取：`data/exercises.mw.js`（`exercises.js` 已接入）
- 本地缓存目录：`.cache/mw-api/`（已在 `.gitignore` 忽略）
- 可通过 `MW_CACHE_DAYS` 调整缓存天数，例如 `MW_CACHE_DAYS=7`

> 说明：免费档是每月 500 次调用。该脚本在离线生成阶段消耗额度，前端运行时不调用 API。

## 技术栈

- **纯原生**：HTML / CSS / JavaScript（ES Modules），无框架，无构建工具
- **数据**：localStorage 持久化，无后端
- **视频**：MuscleWiki 公开 MP4（`autoplay loop muted playsinline`）
- **PWA**：Web App Manifest + Service Worker
- **部署**：GitHub Pages
